一個人 一台電腦 一個檔案

work.txt  (要把內容想好想具體!!)

edit / save(overwrite)

沒辦法拿回以前的版本. (為什麼想拿以前的?? 要舉例!)

需求: 為了要能拿回以前的版本, 解決方法: copy

(有沒有動機想知道兩版之間修改的原因??)

copy work.txt to work_old.txt
copy work.txt to work2.txt
copy work.txt to work_02.txt
copy work.txt to work_20070628.txt
copy work.txt to work_20070628_04_44.txt
copy work.txt to work_20070628_04_44_some_short_description.txt

問題:
檔案多
為了排序 =>
 1. 檔名加上序號 => 將來備份時如果用簡單的 copy, 日期會不見
 2. 檔名有日期加上時間或序號
沒有描述, 不知道兩個版本間的差異是為了什麼而改 => 挑錯個檔案
改名不易, 要許多檔案一起改
麻煩, 因此不會常常另存新檔 => 想要的版本沒被存到

(強弱順序?)

一個人 一台電腦 多個檔案

舉 css 配 html 的例子
舉程式的例子
需求: 必須能挑出檔案 A 的這個版本配檔案 B 的那個版本 才能正常運作
(有沒有不是技術的, 多檔間又有相依關係的好例子?)


解法: 整個目錄 copy / 壓縮

問題:
(include 舊的問題)
不易從檔案樹中知道這次改了哪些檔案(除非看時間, 或用diff工具)
常常連沒有改動的檔案也一起包進去了, 浪費空間

一個人 多台電腦 多個檔案

需求: 兩台或更多台機器更改同一組檔案, 希望兩邊都能有最新的版本(?)

解法1: 網路/中間第三台機器間接傳檔, 不斷的 copy
問題:
1. 有時沒有網路
2. 要花力氣才能確定現在這邊這份是最新的才能開始工作,
   或後來花更多力氣互相挑選最新的檔案 => 失手忘了 copy 或挑了舊的

解法2: 只有一份在其中一台上, 或第三台 server 上
問題: 有時沒有網路, 或慢

解法3: 只有一份在隨身碟上
問題: 空間(太弱) / 忘了帶 / 弄丟 / 壞掉
